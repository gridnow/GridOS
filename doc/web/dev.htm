CPU中断异常处理框架

CPU的中断与异常处理流程。

1，硬件中断控制器描述符以及特定型号的中断控制器操作方法。

2，特定CPU的中断处理表和处理函数。

3，抽象的中断处理流程和中断处理函数的管理（为驱动程序提供相应接口）。

4，特定CPU的异常与内核的异常处理框架对接。

1，完成平台无关的中断注册、中断处理流程，目前在x86-32上测试通过。还缺少中断处理函数的反注册，多处理器中断问题。

2，给驱动程序提供的中断处理函数注册接口还未提供。
多CPU管理框架 探测和发现多CPU，启动和停止多CPU，与进程调度器的结合，等其他与多CPU相关的处理。  
设备管理器 设备和驱动程序描述符的管理、操作。以及设备和驱动所用到的资源（内存资源IO资源）的管理。（注册设备，注册驱动，注册设备所拥有的资源，以及他们的被系统其他应用单位的对接口）。 1。完成了设备对象的注册封装。该封装是对设备最抽象的描述，其中主要包括2类信息的结合：设备管理器的设备信息与驱动程序对于设备的详细描述。详细描述用于具体驱动程序，高层信息用于设备管理器内部。
PCI总线驱动 在设备管理器提供的管理框架中，支持平台无关的PCI总线的设备扫描、使能、资源操作接口。 1，上层代PCI设备扫描码部署完毕，并与设备管理器对接完成。支持的硬件平台可以有很多种，目前是x86。许多其他依赖接口还未编写完成，没有测试。
简单控制台封装 实现系统调试。

二，本阶段主要目标

    1，内核具备分页机制的虚拟内存，支持多线程运行，在多CPU上能进行简单的负载均衡,识别到PCI硬件。

    2，支持虚拟文件系统。

    3，支持基本libc编程接口，可操作文件 。

    4，支持简单控制台。
